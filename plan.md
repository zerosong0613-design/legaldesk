## Phase 1.5 — 구조 개편 + 대시보드 강화

### 핵심 변경: 사건 / 자문 2파트 분리

**사건** (소송): 기존 구조 유지
- 사건번호, 법원, 기일, 상대방 등
- 상태: 접수 → 진행 → 종결 / 보류

**자문**: 소송이 아닌 법률 상담/검토 업무
- 자문 의뢰인, 자문 주제, 자문 유형
- 상태: 접수 → 진행 → 완료 / 보류
- 기일 없음 (대신 마감일)
- Calendar 연동은 마감일 기준

### Step 7. 데이터 구조 확장
**브랜치**: `feat/data-restructure`

**cases.json 변경**:
```json
{
  "version": "2.0",
  "cases": [...],        // 소송 사건
  "consultations": [...]  // 자문
}
```

**자문 데이터 구조**:
```json
{
  "id": "C-2026-001",
  "clientName": "주식회사 테크",
  "clientEmail": "contact@tech.co",
  "clientPhone": "02-1234-5678",
  "category": "자문",
  "type": "계약검토",
  "subject": "공급 계약서 리뷰",
  "status": "진행",
  "deadline": "2026-04-10",
  "fee": null,
  "tags": ["계약", "공급"],
  "openedAt": "2026-03-20",
  "closedAt": null,
  "driveFileId": "...",
  "driveFolderId": "...",
  "lastActivityAt": "..."
}
```

**자문 유형**: `"계약검토"` | `"법률의견서"` | `"규정자문"` | `"일반상담"` | `"기타"`

**구현 내용**:
1. consultations 인덱스 추가 (cases.json v2)
2. caseStore 확장 → consultation CRUD 액션 추가
3. Drive 폴더 구조에 `consultations/` 추가
4. 기존 사건 데이터 마이그레이션 (v1 → v2)
